## 1. 产品概述

### 1.1 产品名称
**MacScreenshot** - Mac原生轻量截图标注工具

### 1.2 产品定位
- 轻量：不占用过多系统资源
- 原生：使用Swift和AppKit原生开发，不基于Electron等跨平台框架
- 专注：只做截图和标注，不添加冗余功能
- 快速：启动快，响应迅速

### 1.3 解决的问题

| 现有问题 | MacScreenshot解决方案 |
|----------|----------------------|
| 第三方截图工具臃肿，启动慢 | 原生轻量，瞬间启动 |
| 很多工具带广告、会员制 | 完全免费开源，无广告 |
| 功能太多太复杂，日常只用截图标注 | 只保留核心功能，界面简洁 |
| 占用内存大，影响系统性能 | 常驻内存仅几MB |
| 快捷键冲突、自定义困难 | 全快捷键可自定义 |

## 2. 目标用户

### 2.1 主要用户
1. **软件开发者** - 需要截图标注Bug、演示功能
2. **内容创作者** - 需要制作教程、文档配图
3. **产品经理** - 需要标注原型、需求文档
4. **设计师** - 需要标注设计稿反馈
5. **普通用户** - 日常需要简单截图标注

### 2.2 用户场景
- 写技术博客时截图标注代码和界面
- 汇报工作时标注数据图表
- 反馈Bug时框选问题区域
- 制作教程步骤示意图
- 快速分享屏幕内容

## 4. 交互设计

### 4.1 主流程

```
用户按下快捷键 → 进入截图模式 → 用户选择区域 → 进入标注模式 → 用户完成标注 → 复制/保存/分享 → 返回系统
```

### 4.2 截图模式交互
1. 按下快捷键后，屏幕变暗提示选择区域
2. 用户拖拽选择区域，实时显示选区大小
3. 松开按键自动进入标注模式
4. ESC 键取消截图，返回正常状态

### 4.3 标注模式交互
1. 顶部/底部工具栏显示所有标注工具
2. 用户选择工具后即可在截图上操作
3. 底部工具栏：撤销 → 重做 → 清空 → 取消 → 完成

### 4.4 快捷键设计

| 操作 | 标注模式快捷键 |
|------|----------------|
| 完成标注 | `Enter` / `Cmd + Enter` |
| 取消 | `Esc` |
| 撤销 | `Cmd + Z` |
| 重做 | `Cmd + Shift + Z` |
| 清空所有 | `Cmd + Shift + Del` |
| 选择矩形工具 | `R` |
| 选择椭圆工具 | `E` |
| 选择箭头工具 | `A` |
| 选择文字工具 | `T` |
| 选择画笔工具 | `B` |
| 选择马赛克工具 | `M` |
| 选择裁剪工具 | `C` |

## 5. 非功能需求

### 5.1 性能要求
- **启动时间**: 从按下快捷键到截图界面显示 < 100ms
- **内存占用**: 后台常驻 < 10MB
- **截图响应**: 选择区域后立即显示标注界面，无明显卡顿
- **保存速度**: 保存大图片 < 1秒

### 5.2 兼容性
- 支持 macOS 12.0+
- 支持 Apple Silicon (M1/M2/M3)
- 支持 Intel 芯片
- 支持 Retina 屏幕

### 5.3 可用性
- 界面简洁，新手可快速上手
- 所有操作都有快捷键
- 错误处理友好，崩溃自动恢复

### 5.4 可维护性
- 代码模块化，易于扩展
- 清晰的代码注释
- 单元测试覆盖核心逻辑
